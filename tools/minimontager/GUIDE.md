# МОНТАЖЕР 1.0 — Гайд

### Йо, шо це таке?

Коротко: ти кидаєш AI озвучку + згенеровані відеокліпи — а воно тобі збирає готове відео для YouTube. Автоматично. Без Premiere. Без After Effects. Без страждань.

Весь пайплайн — від сценарію до фінального рендеру — крутиться через Claude Code + купу AI-сервісів.

---

## Як це працює? Пайплайн по кроках:

### КРОК 1: Сценарій
Claude Code пише сценарій англійською (бо озвучка через ElevenLabs, а він англійську тягне найкраще). На виході — `output/scenario.txt`, чистий текст без розмітки.

### КРОК 2: Промпти для картинок
Сценарій розбивається на сцени. Для кожної сцени Claude пише детальний промпт для **Nanobanana** (AI-генератор картинок). Один промпт = одна картинка = один кадр майбутнього відео.

Промпти лежать тут:
```
prompts/
├── block_1_2.txt          # промпти блоків 1-2
├── block_3_4_5.txt        # промпти блоків 3-4-5
└── prompts_neyrobanan.txt # ВСІ промпти в одному файлі (177 штук)
```

Формат: один промпт на абзац, між промптами — порожній рядок. Слово **"Author"** — це плейсхолдер, куди підставляється твій фейс-референс.

### КРОК 3: Генерація картинок
Кидаєш промпти в **Nanobanana** — отримуєш 177 картинок.

### КРОК 4: Анімація
Картинки анімуються через **Veo3** (Google). Кожна картинка → 8-секундний відеокліп, 1280x720.

### КРОК 5: Озвучка
Сценарій озвучується через **ElevenLabs**. На виході — MP3 файл (~17 хвилин).

### КРОК 6: Автомонтаж (ось тут магія)
Скрипт `scripts/montage.py` бере:
- 177 відеокліпів з `input/clips/`
- озвучку з `input/voiceover.mp3`
- маппінг сцен з `project.json`

І робить:
1. Визначає тривалість аудіо через ffprobe
2. Рахує слова в кожній сцені
3. Розподіляє час **пропорційно кількості слів** (більше слів = довший кліп)
4. Тримає/лупить кожен кліп під потрібну тривалість (FFmpeg)
5. Склеює все в один відеоряд
6. Накладає озвучку
7. Profit: `output/final_video.mp4`

```

│
├── project.json              # Головний конфіг: 177 сцен, кліпи, текст

├── input/
│   ├── voiceover.mp3         # Озвучка (ElevenLabs, ~17 хв)

│
├── output/
│   ├── final_video.mp4       # ФІНАЛЬНЕ ВІДЕО
│   ├── scenario.txt          # Сценарій (чистий текст)
│   └── storyboard.txt        # Сторіборд (опис сцен)
│
├── prompts/
│   ├── prompts_neyrobanan.txt  # Всі 177 промптів для Nanobanana
│   ├── block_1_2.txt           # Промпти блоків 1-2 (65 шт)
│   └── block_3_4_5.txt         # Промпти блоків 3-4-5 (112 шт)
│
└── scripts/
    └── montage.py            # Скрипт автомонтажу (Python + FFmpeg)
```

## project.json — серце проекту

```json
{
  "resolution": "1280x720",
  "audio_file": "input/voiceover.mp3",
  "clips_dir": "input/clips",
  "output_file": "output/final_video.mp4",
  "scenes": [
    {
      "id": 1,
      "clip": "001-video.mp4",
      "text": "This thing weighs as much as a truck..."
    }
  ]
}
```

Кожна сцена: `id` (номер) + `clip` (файл кліпу) + `text` (що говорить диктор в цей момент). Текст використовується для розрахунку тривалості кліпу.

## Як запустити монтаж

```bash
cd "Монтажер 1.0"
python3 scripts/montage.py project.json
```

Потрібно: Python 3, FFmpeg (`brew install ffmpeg`).

## Лайфхаки

- **"Author"** в промптах — плейсхолдер для обличчя. Підстав свій референс перед генерацією.
- Якщо динозаври занадто мультяшні — заміни "Photorealistic CGI" на "Indistinguishable from real footage, practical effects look, BBC nature documentary quality".
- Якщо кадри не збігаються з озвучкою — проблема в тому, що пропорційний розподіл не враховує паузи диктора. Для ідеальної синхронізації потрібен Whisper з word-level timestamps.
- Кліпи по 8 секунд. Якщо сцена довша — кліп лупиться (повторюється). Якщо коротша — обрізається.
